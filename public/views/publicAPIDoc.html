Introduction


We will show all the Public APIs here.

The APIs support GET/POST requests .

You can request by a rate limit of 3 requests/sec , otherwise you will be blocked.

Each API return data in format like :{"status":1, "message":"OK", "result":""}.
status: 1 mean Success , 0 mean Fail
message: OK or fail reason
result: json format data about the request result


Both http and https are supported

Contract APIs


Newly verified Contracts are synced to the API servers within 5 minutes or less

Get Contract ABI for Verified Contract Source Codes 

  • https://etzscan.com/publicAPI?module=contract&action=getabi&address=0xc01fbdf458041359a8eeaf03d9991dd8fd8dbd6f&apikey=YourApiKeyToken

A simple sample for retrieving the contractABI using Web3.js and Jquery to interact with a contract

var Web3 = require('web3');
    var web3 = new Web3(new Web3.providers.HttpProvider());
    var version = web3.version.api;

    $.getJSON('https://etzscan.com/publicAPI?module=contract&action=getabi&address=0xc01fbdf458041359a8eeaf03d9991dd8fd8dbd6f', function (data) {
        var contractABI = "";
        contractABI = JSON.parse(data.result);
        if (contractABI != ''){
            var MyContract = web3.eth.contract(contractABI);
            var myContractInstance = MyContract.at("0xc01fbdf458041359a8eeaf03d9991dd8fd8dbd6f");
            var result = myContractInstance.getBuyPrice();
            console.log("result : " + result);
        } else {
            console.log("Error" );
        }
    });

Event Logs


[Beta] The Event Log API was designed to provide an alternative to the native eth_getLogs. Below are the list of supported filter parameters:

  • [require Parameters] address: contract address
  • [require Parameters] fromBlock: logs in block is great than or equle fromBlock
  • [Optional Parameters] toBlock: logs in block is little than or equle toBlock
  • [Optional Parameters] topics: topics filter. multi topic separate with comma.
    such as: ...&topics=value0,,value2&... this exprission will fetch logs that topic0 is value0 and topic2 is value2.
    each value's length is 66(contain:0x)
  • [Optional Parameters] topic_oprs: topics operation. multi operation separate with comma.
    this parameters explain the relation between the topics above.
    such as: ...&topic_oprs=0_and,1_2_or... this exprission means that topics0=value0 && (topics1=value1 || topics2=value2).
    if there is not topic_oprs, then all topics will connect with 'and'.
  • [Optional Parameters] data: log data(not index param data)
  • [Optional Parameters] returnFilters: return specific fields(full fields as default). multi topic separate with comma.
    such as: ...&returnFilters=address,timestamp,topics,data&...
    All fields as follow:
    address: String (contract address)
    txHash: String (hash of transaction which contain the log)
    blockNumber: Number(blockNumber of block which contain the log)
    timestamp: Number
    from: String(from of transaction)
    to: String(to of transaction)
    logIndex: Number,
    topics: Array ([topic0, topic1, topic2])
    data: String (data of transaction)
    gasUsed:Number
    gasPrice: Number
fromBlock and toBlock accepts the blocknumber (integer, NOT hex) or 'latest' (earliest & pending is NOT supported yet)
  • https://etzscan.com/publicAPI?module=logs&action=getLogs&address=0x013b6e279989aa20819a623630fe678c9f43a48f&&fromBlock=6601088&topic_oprs=0_and,1_2_or&topics=0xddf252ad1be2c89b69c2b068fc378daa952ba7f163c4a11628f55a4df523b3ef,0x000000000000000000000000D0011bC3b4b2354517529fEF2310E1b5326E0c89,0x000000000000000000000000D0011bC3b4b2354517529fEF2310E1b5326E0c89&returnFilters=timestamp,topics,-_id&apikey=YourApiKeyToken

Geth/Parity Proxy APIs


The following are the limited list of supported Proxied APIs for Geth available through etzscan. For the list of the parameters and descriptions please see https://github.com/ethereum/wiki/wiki/JSON-RPC. Parameters provided should be named like in the examples below. For compatibility with Parity, please prefix all hex strings with "0x"

eth_blockNumber
Returns the number of most recent block

  • https://etzscan.com/publicAPI?module=proxy&action=eth_blockNumber&apikey=YourApiKeyToken


eth_getBlockByNumber
Returns information about a block by block number.

  • https://etzscan.com/publicAPI?module=proxy&action=eth_getBlockByNumber&blockNumber=666&apikey=YourApiKeyToken


eth_getBlockTransactionCountByNumber
Returns the number of transactions in a block from a block matching the given block number

  • https://etzscan.com/publicAPI?module=proxy&action=eth_getBlockTransactionCountByNumber&blockNumber=666&apikey=YourApiKeyToken


eth_getTransactionByHash
Returns the information about a transaction requested by transaction hash

  • https://etzscan.com/publicAPI?module=proxy&action=eth_getTransactionByHash&txhash=0x1e2910a262b1008d0616a0beb24c1a491d78771baa54a33e66065e03b1f46bc1&apikey=YourApiKeyToken


eth_getTransactionByBlockNumberAndIndex
Returns information about a transaction by block number and transaction index position

  • https://etzscan.com/publicAPI?module=proxy&action=eth_getTransactionByBlockNumberAndIndex&blockNumber=1267471&index=0x0&apikey=YourApiKeyToken


eth_getTransactionCount
Returns the number of transactions sent from an address

  • https://etzscan.com/publicAPI?module=proxy&action=eth_getTransactionCount&address=0x2910543af39aba0cd09dbb2d50200b3e800a63d2&tag=latest&apikey=YourApiKeyToken


eth_sendRawTransaction
Creates new message call transaction or a contract creation for signed transactions

  • https://etzscan.com/publicAPI?module=proxy&action=eth_sendRawTransaction&hex=0xf904808000831cfde080&apikey=YourApiKeyToken
    • (Replace the hex value with your raw hex encoded transaction that you want to send.
    Send as a POST request, if your hex code is particularly long)


eth_getTransactionReceipt
Returns the receipt of a transaction by transaction hash

  • https://etzscan.com/publicAPI?module=proxy&action=eth_getTransactionReceipt&txhash=0x1e2910a262b1008d0616a0beb24c1a491d78771baa54a33e66065e03b1f46bc1&apikey=YourApiKeyToken


eth_call
Executes a new message call immediately without creating a transaction on the block chain
